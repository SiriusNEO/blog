关于本站

本站为博主的个人博客，名字 Metric Space 取自数学中的度量空间。取这个名字的原因大概有：

  • 博主很喜欢数学，特别是测度论这一块
  • “度量空间” 名字中的 ”空间“ 使其本身很适合作为一个博客的名称（类比 QQ 空间？）。
  • ”度量“ 二字其实也有一些意思。字面上来看，它指的是一个类似距离的概念。在世界上，人与人之间都是存在距离的，用 EVA 中的词来说就是「心之壁」。而当你来访问我的博客时，虽然我们之间存在距离，但至少不是无穷远——因为你通过阅读我的文章，对我有了非负的了解。这也是 ”度量空间“ 这个名字作为博客的一个美好寓意。
  • 我以前很喜欢一些有意思的域名，也因此在机缘巧合下买下了 me.tric.space 这个域名。一开始其实它被用作我的学术主页，后来觉得更适合作为博客的名字。

这个站经历了一次搬迁。之前它还不叫 Metric Space，而是有一个有点文青（小学生）的名字：”没有风经过的旅店“。原先它部署在我的腾讯云 Lighthouse Wordpress 镜像，主题是 WP 中免费的 Akina，建站于 2020 年末，等一系列手续通过之后与2 021 年 1 月 14 日上线（即第一篇 Hello World 发表的时间）。

后面它换成了 Github Pages + Hexo 的方案（也就是现在的样子），名字也改成了 Metric Space。主题换成了 Hexo 的 Cactus。搬迁于 2023 年 10 月 9 日，差不多隔了快 3 年吧。

我会不定期（有时可能几个月都不更新）在这个地方发一些关于学习生活的文章，以及从别的创作平台（知乎、豆瓣等）搬运我的文章到这里。这个站的文章暂时不打 Tag，而是用 Category 进行分类。我用游戏《东方 Project》里的角色 ”纯狐“ 的符卡名来为不同的文章分类命名，详情请查看主页的 Category 一栏。

关于我

博主目前在上海某高校的 CS 专业本科就读，研究生预计会去北京某高校就读计算机系统与体系结构方向的 CS PhD。关于学术方面，请查看我的学术主页。研究方向主要是计算机系统，由于做的很多是和机器学习交叉的领域，所以对一些机器学习算法也比较了解。对理论计算机处于听过课的状况，对 PL 和一些数学属于业余爱好者水平。

  • 平常开发使用最多的 IDE 是 Visual Studio Code，常用的计算机语言是 C++ 和 Python，对偏软件方向的 Go，Web 开发方面的 JavaScript 和偏硬件方面的 Verilog 也比较了解。会非常主动地去创作自己觉得有意思的东西。Github 主页上主要有一些课程项目、科研项目以及一些个人闲暇时候写的小东西。

  • 习惯使用 Markdown 来记各种各样的笔记。有时会使用 Notion 来完成不同设备之间的云同步，以及更加方便地分享给别人看。

  • 通常使用 Windows 和 MacOS 双操作系统。在 Windows 上我一般使用 WSL（The Windows Subsystem for Linux）来获得一个贴近 Linux 的开发环境。喜欢苹果生态的协同。
  • 对编译器方面挺感兴趣，曾经因为课程写过一个 Toy 的编译器，并且在这个过程中读了很多关于编译优化的书和理论。时常幻想自己能够发明一个新的编程语言。
  • 曾经有过数学竞赛和信息学奥林匹克竞赛（OI）经历，了解过一些简单的算法。

我的兴趣爱好比较窄而且都偏小众，可能运动方面热爱会多一点？跑步、足球、乒乓球都比较喜欢，且水平还可以。声乐反正是一窍不通。
